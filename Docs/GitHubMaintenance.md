# GitHub maintenance

Обновлено: 2026-07-02.

Источник проверки: текущий checkout и `origin` (`https://github.com/wssdozh/oreshkes.git`). GitHub-приложение Codex не имеет доступа к репозиторию, поэтому PR и настройки репозитория через API не проверялись.

## Принятое решение

Актуальная основная линия проекта - `backup/create-fire-executor-post-lfs-rewritten`.

Остальные ветки считаются второстепенными. Их всё равно не нужно удалять вслепую: сначала лучше выровнять `main` на актуальный коммит, оставить страховку старого `main`, а уже потом отдельно решить, какие ветки можно удалить.

## Безопасный порядок уборки

1. Ничего не удалять.
2. Создать страховку старого `main`, например `backup/old-main-before-main-realign`.
3. Выровнять `main` на коммит `backup/create-fire-executor-post-lfs-rewritten`.
4. После проверки GitHub включить защиту `main`.
5. Старые ветки удалять только отдельным списком и после явного подтверждения.

## Предлагаемые правила веток

- `main` - стабильная основная ветка.
- `feature/<name>` - новые игровые фичи.
- `fix/<name>` - исправления.
- `chore/<name>` - документация, настройки, housekeeping.
- `backup/<name>` - временные страховочные ветки после миграций или LFS-переписывания.

## Перед удалением ветки

Проверить:

- нет ли открытого PR из этой ветки;
- все нужные коммиты уже есть в основной ветке или осознанно отклонены;
- есть тег или backup-ветка, если ветка нужна как страховка;
- удаление подтверждено владельцем проекта.

Команды удаления специально не приводятся здесь как рабочая инструкция: их лучше выполнять только после финального списка веток и отдельного подтверждения.
